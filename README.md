# Clinical AI Safety Monitor (CASM)

Clinical AI Safety Monitor is an experimental project for checking clinical claims generated by LLMs against external medical evidence.

## What the project does

CASM is designed as a 2-stage safety pipeline:

1. **Claim extraction**  
   Convert an LLM response into atomic, structured clinical claims.
2. **Knowledge verification**  
   Validate verifiable claims using PubMed evidence, NLI classification, and OpenFDA adverse-event signals.

## Current project status

This repository is **in active development** and currently **prototype-first**:

- Core stage files exist under `pipeline/` and `clients/`.
- Main runnable workflow is still notebook-based in `experiments/`.
- Several imports referenced by pipeline modules (`utils`, `classifiers`) are not yet present in this repo.
- `main.py` is currently empty.
- No automated test suite is currently set up.

## Key components

### Claim Extractor (`pipeline/claim_extractor.py`)
- Hybrid NLP extraction using Med7 + scispaCy.
- Splits text into atomic claim sentences.
- Classifies claims into medical claim types.
- Produces structured `Claim` objects with entities, drugs, dosages, and conditions.

### Knowledge Verifier (`pipeline/knowledge_verifier.py`)
- Retrieves semantic matches from Chroma.
- Fetches PubMed abstracts when local evidence is insufficient.
- Runs NLI classification over evidence.
- Queries OpenFDA for adverse-event counts.
- Produces structured `EvidenceResult` outputs.

### Data models (`models/`)
- `Claim`, `ClaimType`, `Entity`
- `EvidenceResult`, `SearchResult`, `NLIVerdict`

### External clients (`clients/`)
- `OpenFDAClient` for adverse-event counts.
- `PubMedFetcher` for live abstract retrieval.
- `ChromaStore` for embedding-based semantic retrieval.

## Setup

### Option 1: pip + requirements

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
python -m pip install --upgrade pip
pip install -r requirements
```

### Option 2: uv + pyproject

```bash
uv sync
```

> Note: This project uses biomedical NLP models (Med7/scispaCy/Sentence-Transformers). You may need additional model downloads depending on which workflow you run.

## How to run now

At the moment, the easiest way to explore functionality is through notebooks:

- `experiments/CASM.ipynb` → claim extraction prototype
- `experiments/test.ipynb` → deterministic verification prototype
- `experiments/biobert_test.ipynb` → biomedical NLI experiment
